1. How to Use
Once installed AutoReview will be in your extension menu or pinned to the settings area in the top right of the browser. Clicking on the icon will open the AutoReview window.

Clicking ‘SELECT EXPORT’ will open a windows explorer to select the Power Automate export (this should be a zip file)
a. Import legacy export
Legacy Exports are only a single flow, and can be reviewed by selecting the export zip. As only 1 flow is included you will see a label (name label) stating the name of the Flow or the reason it failed. To view the report click ‘OPEN REPORT’
b. Import Solution export
​Solution exports act the same way as legacy exports, unless there are multiple flows contained within the solution. If there are then a list of flows will appear, simply click the flow you want to review (the name label will update with the selected flow)

c. Load Previous
​The last flow you uploaded is stored in our browser cache. To reload it click the 'Load Last Flow' option from the 'SETTINGS' menu

d. Shortcut Keys
ALT+A Open AutoReview Window
ALT+L Load Previous
ALT+R Open Review
ALT+E Open Report
ALT+D Open Diagram
ALT+C Clear Last Saved
e. Limitations
The flow definition file has had multiple updates from Microsoft. Older exports may have an older schema, which does not include operationMetadataId’s. Without operationMetaId’s we can not extract the actions from the flow definition, so these will fail the review process. If the flow is imported back into Power Automate, edit and export again they should be updated to the latest schema.

2. Review File

a. Review
The review has 8 sections:
  • 4 Score cards - Red/Amber/Green
  • Overall Score - Including warnings, failures and status of key checks
  • Trigger - Details like type and parameters
  • Variables - With ✖ for fails and ⚠ for warnings
  • All actions - With key details
  • API - Actions that call external data
  • Exceptions - What actions have run after Failed
  • Connection References
​
b. Filters
For Variables, Actions & API the tables can be filtered by inputting search term and selecting column to filter

3. Additinal Features

a. Report
​​The Report is very similiar to Review, with all scoring and checks removed. The report is designed to support with documention
b. Diagram
A visual representation of the flow can been seen in the diagram section. Currently this doesn't match the Power Automate view 1 to 1, with the main difference being container steps (e.g Scope or Condition) In Power Automate all child actions are inside, in AutoReview they follow after, with a dotted line the next action after the container
From the diagram clicking on action will open a popup with key information about the action.
​Additionall a SVG copy can be downloaded by clicking the download icon in the right bar
c. View definition
The definition file is the json file that contains all the flow details and is extracted from the export zip. If you wish to view it you can click the ‘DEFINITION' button.​

d. View data
Additional to the report and definition you can also see the data export json by clicking the ‘DATA’ button. The data is the information extracted from the definition file and used to populate the report. You can use this if you want to create your own solution/report process.
e. Download CSV
To view all fields the option to download:
  • Actions
  • Variables
  • Connections 

​in CSV format is available
4. Bug Report
To help improve the performance of AutoReview we need an extensive range of flows to test against. Any flows that fail will be very useful for use to test against and fix. By clicking the ‘Report Bug’ option from the ‘SETTINGS MENU’ will open an email to send the flow definition that failed. Please only send if you can remove any personal data you don't want to share with us. Any additional comments on the bug would also be helpful. The Definition can be copied from the window opened by the DEFINITION' button.
5. Review Standards
AutoReview not only extracts key information about the flow but also grades it against set requirements.

a. Guide
​Multiple aspects of the flow are looked at in regards to it meeting AutoReview’s expected levels, this covers everything from count of actions to naming conventions, for more information and what is looked at please read this article Power Automate- Code Reviews.
​b. Warnings

Warnings are highlighted by orange warning triangles, these are used to highlight possible issues, but where AutoReview can’t be sure. An example would be the API Actions table, where a action will be flagged as a warning if it doesn’t have:
  • A filter query
  • Pagination
  • Retry
As in most cases these should be used, but in some cases, and some connectors they may not be required. Please review then and ignore them if you are confident it is correct.
c. Errors 
Warnings are highlighted by red cross (please note not all errors are highlighted, only ones visible in tables). In the above example the variable ‘iCounter’ does not meet the naming convention requirement. All errors should be corrected before promotion to production.

d. Filters

For both the Actions and API tables there is a filter input in the top right corner. The actions will be filtered by any name that contains the input text.

6. Config own Standards

The option to add your own grading by uploading a config file, this will set:
  • Overall point scoring
  • Action,Complexity,Variable and Connection Red/Amber/Green
  • Action Complexity scores
  • Variable naming letter/s
Click ‘Download Config Templates’, a zip file should download to your computer. The zip contains 6 files (4 json config files and 2 README files).
The readme files give detailed guy on how to edit the config json files.
​
Once edited they need to be zipped again and uploaded like a flow export. Any config files included in the zip will be uploaded, you should see a confirmation line for each.

The sReference you input will appear in the top right corner of your reports.

The sReference you input will appear in the top right corner of your reports.

To restore back to defaults click ‘Restore Default Configs’
7. Print Review/Report or Download
The print review or report you can use the web browser print function (this can also save as a pdf). The best layout for printing is
  • A3 Page size
  • Landscape
  • Print to Colour
Review, Report and Diagram can now be saved locally as a fully working web page. Just click the download button in the side/top menu.
